## [2026-04-16] ログイン後のデータロード不具合修正

### 目的

ログイン後に「コンティニュー」を選択してもセーブデータが読み込まれない不具合を修正する。

### 問題の背景

以下の2つのバグが原因でセーブデータが正しく扱われていなかった。

**バグ1：`hasSave()` の非同期処理の誤用**

`js/ui.js` の `renderTitleScreen()` 内で `hasSave()` を同期的に呼び出していた。
`hasSave()` は `async` 関数なので Promise を返す。
テンプレートリテラル内で Promise は常に truthy と評価されるため、
セーブデータが存在しなくても「コンティニュー」ボタンが常に表示されていた。

```js
// 修正前（バグあり）
${hasSave() ? '<button id="btn-continue">コンティニュー</button>' : ''}
// → hasSave() が Promise を返すので常にボタンが表示される
```

**バグ2：ログイン直後の userId 取得タイミング問題**

`signIn()` でログインした直後、`afterLogin()` から `loadFromDB()` を呼ぶ際に
`getCurrentUserId()` → `getSession()` と再度セッションを取得していた。
ログイン直後のタイミングによってはセッションが取得できず、
セーブデータが存在しても null が返ることがあった。

### 修正内容

#### `js/supabase.js` — `loadFromDB(userId = null)`

- `userId` を引数で直接受け取れるよう変更
- 引数がない場合は従来通り `getCurrentUserId()` で取得（後方互換維持）

```js
// 変更前
async function loadFromDB() {
  const userId = await getCurrentUserId();
  ...
}

// 変更後
async function loadFromDB(userId = null) {
  const uid = userId || await getCurrentUserId();
  ...
}
```

#### `js/state.js` — `loadGame(userId = null)` / `hasSave(userId = null)`

- `loadGame()` と `hasSave()` に `userId` 引数を追加
- 内部の `loadFromDB()` 呼び出しに `userId` を渡すよう変更

#### `js/main.js` — `onLoginSuccess(session)` / `afterLogin(userId)`

- `onLoginSuccess` が `session` オブジェクトを受け取るよう変更
- `signIn()` が返す session から userId を直接取得し `afterLogin` へ渡す
- `renderTitleScreen()` が async になったため `await` を追加
- ページリロード時（既存セッションあり）も `session.user.id` を渡すよう修正

```js
// 変更前
window.onLoginSuccess = async function() {
  showLoading('セーブデータ読み込み中...');
  await afterLogin();
};

async function afterLogin() {
  const savedState = await loadGame();
  ...
  renderTitleScreen();  // awaitなし
}

// 変更後
window.onLoginSuccess = async function(session) {
  showLoading('セーブデータ読み込み中...');
  await afterLogin(session?.user?.id ?? null);
};

async function afterLogin(userId = null) {
  const savedState = await loadGame(userId);
  ...
  await renderTitleScreen(userId);  // awaitあり
}
```

#### `js/ui.js` — `renderTitleScreen(userId = null)` / `renderLoginScreen()`

- `renderTitleScreen()` を `async` 関数に変更
- `hasSave(userId)` を `await` で正しく待機してからボタンを描画
- セーブデータ確認中はスピナーを表示
- `signIn()` の戻り値から `session` を取得して `onLoginSuccess` に渡すよう修正

```js
// 変更前
function renderTitleScreen() {
  app.innerHTML = `...${hasSave() ? '<button>コンティニュー</button>' : ''}...`;
}

// 変更後
async function renderTitleScreen(userId = null) {
  // スピナー表示
  app.innerHTML = `<div>...</div>`;
  // セーブ確認を正しくawait
  const hasSaveData = await hasSave(userId);
  app.innerHTML = `...${hasSaveData ? '<button>コンティニュー</button>' : ''}...`;
}
```

### 修正後の動作

| ケース | 修正前 | 修正後 |
|--------|--------|--------|
| セーブなしの新規ユーザー | コンティニューが常に表示 → 押すとエラー | コンティニューが表示されない |
| セーブありユーザーのログイン | DBロードが失敗することがあった | セッションIDを直接渡してタイミング問題を解消 |
| セーブありでタイトル画面が表示された場合 | コンティニューを押してもエラー | 正しくセーブデータを読み込める |
