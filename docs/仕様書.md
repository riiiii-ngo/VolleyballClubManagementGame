## 1. ゲーム概要

高校バレー部の監督として選手を育成し、日本一（春高バレー優勝）を目指す育成シミュレーションゲーム。  
「私を甲子園に連れてってや・栄冠ナイン」のバレー版をコンセプトとする。

**主な要素**
- 1週単位でゲームが進行（1年 = 48週）
- 練習でパラメータを育成 → 試合で評判・ポイントを獲得 → スカウト・ショップでチーム強化
- プレイヤーデータはSupabase（DB）に保存、ローカルにlocalStorageキャッシュ

## 3. データ管理・保存戦略

### 保存フロー（書き込み）

```
メモリ(G変数) → localStorage（即時・同期） → Supabase（非同期・fire-and-forget）
```

`saveGame(state)` を呼ぶと、まずlocalStorageへ書き込み、並行してSupabaseへ非同期保存する。

### 読み込みフロー

```
Supabase優先 → 失敗時 localStorage フォールバック
```

`loadGame(userId?)` は Supabase から取得を試み、失敗した場合は localStorage のキャッシュを返す。

### マスターデータ

アイテム・設備・練習メニュー・スケジュールは全プレイヤー共通の値として `data/*.json` に保持し、  
`loadMasterData()` で起動時に一括フェッチしてグローバル変数に代入する。

| `matchLog` | object[] | 試合結果ログ |

```
week = monthIndex * 4 + (weekInMonth - 1)
monthIndex: 0=4月, 1=5月, ..., 11=3月
```

## 5. 選手システム

### ポジションと適性パラメータ

| ポジション | 略称 | 伸びやすいパラメータ |
|---|---|---|
| アウトサイドヒッター | OH | スパイク・パワー・サーブ |
| ミドルブロッカー | MB | スパイク・ブロック・スピード |
| オポジット | OP | スパイク・レシーブ・サーブ |
| セッター | Se | レシーブ・トス・テクニック |
| リベロ | Li | レシーブ・テクニック・スピード |

### パラメータ一覧

`spike / receive / block / serve / toss / power / speed / technique / stamina`（9種）

値域: 1〜99（clamp処理あり）

### 選手データ構造

```js
{
  id, name, grade,       // 1〜3年生
  position,              // 'OH'|'MB'|'OP'|'Se'|'Li'
  isAllRounder,          // boolean
  params: { spike, receive, ... },
  currentStamina,        // 0〜maxStamina
  maxStamina,            // 最大100
  potential,             // 1〜3（潜在能力、現状表示のみ）
  rank,                  // スカウト選手のみ 1〜5
}
```

### オールラウンダー

- 全パラメータが均等に伸びやすい（適性ボーナス×1.3 が全パラメータに適用）
- スカウト時にランダム確率で獲得可能（ランダム指定で確率UP）

### 初期チーム構成

13名で開始（3年生5名・2年生4名・1年生4名）。  
statBase は弱小校ベース（22）+ 学年ボーナス（×(grade-1)×5）。

### パラメータ成長ロジック

```
staminaFactor = currentStamina < 10 ? 0.4 : 1.0
statGrowth = 現在値に応じて 0.5〜3（高いほど伸びにくい）
growth = statGrowth × (baseGrowth/2) × efficiency × staminaFactor × posBonus × random(0.8〜1.2)
```

### スタミナ

- 練習で消費（メニューごとに staminaCost が異なる）
- 週次で自然回復 +8
- 試合週は回復 +5
- 試合後消費: `10 + セット数×5 + random(0〜4)`
- 10%以下でパラメータ成長量・試合能力が大幅低下

### 戦闘力スコア（combatScore）

ポジション別の重み付き平均。スタミナ10未満で×0.6、30未満で×0.8。

| ポジション | 計算式 |
|---|---|
| OH | (spike×2 + power + serve + receive) / 5 |
| MB | (spike×2 + block×2 + speed) / 5 |
| OP | (spike×2 + serve + receive + power) / 5 |
| Se | (toss×2 + technique×2 + receive) / 5 |
| Li | (receive×3 + technique + speed) / 5 |

## 7. 試合システム

### スタメン構成

| スロット | ポジション | 人数 |
|---|---|---|
| OH1, OH2 | アウトサイドヒッター | 2 |
| MB1, MB2 | ミドルブロッカー | 2 |
| OP | オポジット | 1 |
| Se | セッター | 1 |
| Li | リベロ | 1 |

計7名（6名コートに入りリベロは後衛交代）。

### ローテーション

初期順: `[OH1, MB1, OP, OH2, MB2, Se]`  
サイドアウト時に時計回りで1つ進む。

**コート内ポジション**
- pos1（バックライト）= サーバー
- pos2〜4（フロント） = スパイク・ブロック参加
- pos5〜6, pos1（バック） = レシーブ参加

**リベロ交代**: バック（pos5, pos6）のMBを自動でリベロと入れ替え。

### ラリーシミュレーション

各ラリーは以下の順で確率判定を行う。

1. **サーブ失敗（フォルト）**: `max(2, 15 - serve×0.1) × staminaMod`
2. **サーブエース**: `serve × 0.12`
3. **レシーブ失敗**: `(30 + receive×0.7 + liberoBonus15) × staminaMod` で成功判定
4. **トス失敗**: `30 + toss×0.7`
5. **スパイク → ブロック**: `avgBlock×0.4 + 人数ボーナス8 - spike×0.15`
6. **スパイク失敗**: `20 + spike×0.8 × staminaMod`
7. **ディグ（レシーブ）**: `15 + receive×0.5 × staminaMod`
8. **ラリー継続→統計ベース決着**: combatScoreの比率で決定

### セットシミュレーション

- 通常セット: 25点先取・2点差必要
- 最終セット（5セットマッチ時）: 15点先取・2点差必要
- 無限ループ防止: 200ラリーで強制終了

### 試合形式

| 試合名 | セット数 |
|---|---|
| 通常 | 3セットマッチ（先2セット取得で勝利） |
| インターハイ決勝・春高予選決勝・春高準決勝・春高決勝 | 5セットマッチ |

### 対戦相手

| 大会 | 1回戦 | 準決勝 | 決勝 |
|---|---|---|---|
| 県大会・春高予選 | 弱小校(35) | 中堅校(55) | 強豪校(75) |
| インターハイ・春高 | 中堅校(55) | 強豪校(75) | 全国強豪(85)/プロ養成(95) 50% |

括弧内は能力値ベース。

### 試合後の報酬

#### 評判ポイント（repPointsWin）

| 大会 | 1回戦 | 準決勝 | 決勝 |
|---|---|---|---|
| 県大会 | +5 | +10 | +20 |
| インターハイ | +10 | +20 | +40 |
| 春高予選 | +10 | +20 | +30 |
| 春高 | +20 | +40 | +100 |

#### ショップポイント（matchPointsWin）

| 大会 | 1回戦 | 準決勝 | 決勝 |

| 県大会 | 30 | 60 | 100 |

## 12. 年度サイクル

week 47（3月4週）終了後に年度末処理が走る。

1. 3年生を卒業（リスト・スタメン・練習グループから除外）
2. 残選手の学年を +1
3. スタメンから卒業生を除外
4. 練習グループから卒業生を除外
5. トーナメント状態をリセット
6. `year++`, `week = 0`

## 13. マスターデータ一覧

### data/schedule.json

- `months`: 月の順序配列（4〜3月）
- `monthNames`: 月番号→表示名
- `matchSchedule`: week番号→試合情報
- `tournamentNames`: 大会ID→表示名
- `repPointsWin`: 勝利時の評判ポイント
- `matchPointsWin`: 勝利時のショップポイント

### data/items.json

- `items`: アイテム定義リスト
- `facilities`: 設備定義リスト

### data/practice_menus.json

- `menus`: 練習メニューリスト（id, name, params, tier, minRep, staminaCost, mini?）
- `growth`: tierごとのbaseGrowth（`{"1":2,"2":4,"3":6}`）
- `minigameGrowth`: ミニゲーム用 baseGrowth（1）

### data/enemies.json

- `teams`: 対戦相手チーム定義（id, name, level, stats）
  - stats: attack, defense, serve, receive, stamina の5値
